# **App Name**: Global Weather View

## Core Features:

- City Configuration: Load city data, including name and country, from a JSON configuration file.
- Weather Data Fetching: Fetch current weather conditions for each city using a weather API based on the loaded configuration. Must take API limits into consideration. Consider an API 'tool' which the app could use for various options for the weather data.
- Weather Display: Display weather information for each city, including temperature, conditions, and other relevant data points.
- City Search: Allow users to search for a specific city from the configured list.
- Unit Selection: Allow users to toggle temperature display between Celsius and Fahrenheit.
- Automatic Refresh: Automatically refresh weather data at a defined interval.

## Style Guidelines:

- Primary color: Sky blue (#70B7FF) to reflect the theme of weather and sky.
- Background color: Light gray (#F0F4F7), providing a neutral backdrop.
- Accent color: Orange-red (#FF6B6B) to highlight interactive elements and important weather details.
- Body and headline font: 'PT Sans', a humanist sans-serif for clear and accessible reading.
- Use weather-themed icons to represent different conditions (sunny, cloudy, rainy, etc.).
- Clean, card-based layout for displaying city weather information. Grid layout to display multiple locations simultaneously.
