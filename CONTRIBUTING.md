# Contributing to the Weather App

Thank you for considering contributing to our Weather App! We welcome contributions from everyone, whether you are fixing a bug, adding a feature, or improving documentation. Here’s how you can help.

## How to Contribute

1. **Fork the Repository**  
   Visit the repository on GitHub (https://github.com/zenika-open-source/how-to-contribute-opensource-react.git) and click on the "Fork" button to create your own copy of the repository.

2. **Clone Your Fork**  
   Clone your forked repository to your local machine:

   ```bash
   git clone https://github.com/your-username/how-to-contribute-opensource-react.git
   cd how-to-contribute-opensource-react
   ```

3. **Create a New Branch**
    Create a new branch for your feature or bugfix:

    ```bash
    git checkout -b feature/my-new-feature
    ```

4. **Make Your Changes**
Implement your changes and write tests to ensure everything works as expected.

5. **Commit Your Changes**
Commit your changes with a descriptive message:

    ```bash
    git commit -m "Add new feature that allows users to view weather details"
    ```

6. **Push to Your Fork**
Push your branch to your fork on GitHub:

    ```bash
    git push origin feature/my-new-feature
    ```

7. **Open a Pull Request**
Go to the original repository and click on the "New Pull Request" button. Select your branch and create the pull request.

**Finding Tasks to Work On**
If you're looking for tasks to contribute, check out the following:

Open Issues: Look at the Issues section of the repository. Issues labeled with good first issue or help wanted are great starting points for new contributors.

Feature Requests: Sometimes, users and maintainers suggest new features. If you see something that interests you, feel free to take it on!

Documentation Improvements: Read through the documentation and see if there's anything that could be clarified or improved.

Code Review: If you have experience, review existing pull requests and provide feedback!

**Tips for Success**
Follow the Coding Standards: Make sure your code follows the existing style and conventions used in the project. This helps maintain consistency throughout the codebase.

Write Clear Commit Messages: Use clear and descriptive commit messages to explain your changes. This helps reviewers understand the context of your modifications.

Test Your Changes: Ensure that your changes don't break existing functionality. Run all tests before submitting your pull request to ensure everything works correctly.

Ask for Help: If you're unsure about something, don't hesitate to ask for help. You can open an issue or contact maintainers directly.

**Code of Conduct**
By participating in this project, you agree to abide by our Code of Conduct. We are committed to providing a welcoming and inclusive environment for all contributors.

**Thank You!**
Your contributions are what make this project great! We appreciate your time and effort in helping us improve the Weather App. Happy coding!
